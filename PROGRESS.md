Full chain of fixes, in order hit:
  1. **Gradle toolchain migration**: `android/settings.gradle` rewritten to the declarative `pluginManagement`/`plugins{}` style (old `apply from: ".../app_plugin_loader.gradle"` script-apply is no longer supported at all). `android/build.gradle` simplified (dropped `buildscript{}`, `jcenter()` → `mavenCentral()`). `android/app/build.gradle` converted to a `plugins{}` block (`com.android.application`, `kotlin-android`, `dev.flutter.flutter-gradle-plugin`), `namespace` added (AGP 8 mandatory). Gradle 7.4 → **8.14.2** (first tried `-all` distribution, it stalled/hung mid-download for 50+ min on this network — switched to the smaller `-bin` distribution, worked immediately). AGP 7.1.2 → **8.11.1**, Kotlin 1.6.10 → **2.2.20** (Flutter's own stated minimums at time of build, not just "newer").
  2. **`plugins{}` block ordering bug**: Gradle requires `plugins{}` to be the first block in a script; had to move it above the `local.properties`-reading logic in `app/build.gradle`.
  3. **`afterEvaluate`-on-already-evaluated-project bug**: added a `subprojects{}` patch in root `build.gradle` to backfill a `namespace` for third-party plugins that predate AGP 8 (e.g. `flutter_native_timezone`) and don't declare one themselves. Applying it to `:app` too crashed ("project is already evaluated") because `:app` is evaluated eagerly via `evaluationDependsOn(':app')` — excluded `:app` from the loop (it already declares its own namespace).
  4. **Core library desugaring**: `flutter_local_notifications` 22.x requires it. Added `coreLibraryDesugaringEnabled true` + `desugar_jdk_libs:2.1.4` to `app/build.gradle`.
  5. **`android:exported` manifest error**: targetSdk bump (30 → `flutter.targetSdkVersion`, resolves to 36) crosses the Android 12 threshold requiring every intent-filtered activity to declare `android:exported` explicitly. Added `android:exported="true"` to `MainActivity` in `AndroidManifest.xml`.
  6. **JVM target mismatch**: old plugins (e.g. `flutter_native_timezone`) don't declare `compileOptions`, defaulting to Java 8, while the bumped Kotlin plugin defaults Kotlin compilation to JVM 17 — extended the same `subprojects{}` patch to force `compileOptions` (Java 17) and Kotlin `compilerOptions.jvmTarget` (17) across every plugin subproject.
  7. **compileSdk too low**: `motion_sensors` pinned its own low `compileSdkVersion`, incompatible with Java 17 source (`>=30` required). Extended the same patch to force `compileSdk 36` (matches `:app`, via `flutter.compileSdkVersion`) on every plugin subproject.
