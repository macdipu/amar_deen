- [x] TASK-003: green build confirmed ‚Äî 2026-07-19, `flutter build apk --debug` succeeded (`build/app/outputs/flutter-apk/app-debug.apk`). This turned out to be a much bigger lift than a normal dependency bump: the Android build config predated Flutter's mandatory Gradle-plugin-loader change entirely (Gradle 7.4, AGP 7.1.2, Kotlin 1.6.10, `jcenter()`), so nothing would build at all, for anyone, regardless of this session's dependency work. Full chain of fixes, in order hit:
  1. **Gradle toolchain migration**: `android/settings.gradle` rewritten to the declarative `pluginManagement`/`plugins{}` style (old `apply from: ".../app_plugin_loader.gradle"` script-apply is no longer supported at all). `android/build.gradle` simplified (dropped `buildscript{}`, `jcenter()` ‚Üí `mavenCentral()`). `android/app/build.gradle` converted to a `plugins{}` block (`com.android.application`, `kotlin-android`, `dev.flutter.flutter-gradle-plugin`), `namespace` added (AGP 8 mandatory). Gradle 7.4 ‚Üí **8.14.2** (first tried `-all` distribution, it stalled/hung mid-download for 50+ min on this network ‚Äî switched to the smaller `-bin` distribution, worked immediately). AGP 7.1.2 ‚Üí **8.11.1**, Kotlin 1.6.10 ‚Üí **2.2.20** (Flutter's own stated minimums at time of build, not just "newer").
  2. **`plugins{}` block ordering bug**: Gradle requires `plugins{}` to be the first block in a script; had to move it above the `local.properties`-reading logic in `app/build.gradle`.
  3. **`afterEvaluate`-on-already-evaluated-project bug**: added a `subprojects{}` patch in root `build.gradle` to backfill a `namespace` for third-party plugins that predate AGP 8 (e.g. `flutter_native_timezone`) and don't declare one themselves. Applying it to `:app` too crashed ("project is already evaluated") because `:app` is evaluated eagerly via `evaluationDependsOn(':app')` ‚Äî excluded `:app` from the loop (it already declares its own namespace).
  4. **Core library desugaring**: `flutter_local_notifications` 22.x requires it. Added `coreLibraryDesugaringEnabled true` + `desugar_jdk_libs:2.1.4` to `app/build.gradle`.
  5. **`android:exported` manifest error**: targetSdk bump (30 ‚Üí `flutter.targetSdkVersion`, resolves to 36) crosses the Android 12 threshold requiring every intent-filtered activity to declare `android:exported` explicitly. Added `android:exported="true"` to `MainActivity` in `AndroidManifest.xml`.
  6. **JVM target mismatch**: old plugins (e.g. `flutter_native_timezone`) don't declare `compileOptions`, defaulting to Java 8, while the bumped Kotlin plugin defaults Kotlin compilation to JVM 17 ‚Äî extended the same `subprojects{}` patch to force `compileOptions` (Java 17) and Kotlin `compilerOptions.jvmTarget` (17) across every plugin subproject.
  7. **compileSdk too low**: `motion_sensors` pinned its own low `compileSdkVersion`, incompatible with Java 17 source (`>=30` required). Extended the same patch to force `compileSdk 36` (matches `:app`, via `flutter.compileSdkVersion`) on every plugin subproject.
  8. **Two genuinely dead plugins** (not a config problem ‚Äî their own Kotlin source referenced `PluginRegistry.Registrar`/`.messenger()`/`.context`, Flutter's removed v1-embedding APIs): `flutter_native_timezone` ‚Üí **`flutter_timezone`** (Dipu's call), `motion_sensors` ‚Üí **`flutter_qiblah`** (Dipu's call, after researching `sensors_plus`/`flutter_compass`/`flutter_qiblah` options). See `DECISIONS.md` for full rationale on both swaps.
  9. **Java heap OOM during Jetifier**: `org.gradle.jvmargs=-Xmx1536M` was too small for AGP 8 + Kotlin 2.x tooling processing multiple large per-ABI Flutter engine jars. Bumped to `-Xmx4608M`, and killed stale Gradle daemons left running from the earlier 7.4/8.10.2/8.14 version attempts.
